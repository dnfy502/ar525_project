\section{Proof of Theorem~\ref{thm:zeromean}}
\label{app:zeromean}

\begin{proof}[Proof of Optimal Action Invariance Under Symmetric Noise]
Let $f : \mathbb{R}^3 \to \mathbb{R}^2$ denote the deterministic landing map, assumed
smooth and invertible near the optimum.
By first-order Taylor expansion around $\mathbf{v}^*$:
\[
  f(\mathbf{v}^* + \boldsymbol{\epsilon}) \approx f(\mathbf{v}^*) + J_f \boldsymbol{\epsilon},
  \qquad J_f = \tfrac{\partial f}{\partial \mathbf{v}}\big|_{\mathbf{v}^*}.
\]
The expected landing position is
\[
  \mathbb{E}_\epsilon[f(\mathbf{v}^* + \boldsymbol{\epsilon})]
  = f(\mathbf{v}^*) + J_f \mathbb{E}[\boldsymbol{\epsilon}]
  = f(\mathbf{v}^*),
\]
since $\mathbb{E}[\boldsymbol{\epsilon}] = \mathbf{0}$.
The noise-averaged cost gradient with respect to $\mathbf{v}$ at $\mathbf{v}^*$ is
\[
  \nabla_{\mathbf{v}} J_{\mathrm{aware}}\big|_{\mathbf{v}^*}
  = J_f^\top \mathbb{E}_\epsilon\!\left[\nabla_{\mathbf{p}} c\bigl(f(\mathbf{v}^*
  + \boldsymbol{\epsilon})\bigr)\right].
\]
At the noiseless optimum $\mathbf{v}^*$ where $f(\mathbf{v}^*) = \mathbf{P}$,
the landing error $f(\mathbf{v}^* + \boldsymbol{\epsilon}) - \mathbf{P} \approx
J_f \boldsymbol{\epsilon}$ is an odd function of $\boldsymbol{\epsilon}$ to first
order.
The cost gradient
$\nabla_{\mathbf{p}} c(\mathbf{p}) = (2(\mathbf{p}-\mathbf{P})/\ell_c^2)\,
\exp(-\|\mathbf{p}-\mathbf{P}\|^2/\ell_c^2)$ is also an odd function of the landing
error $\mathbf{p} - \mathbf{P}$.
Therefore $\nabla_{\mathbf{p}} c(f(\mathbf{v}^* + \boldsymbol{\epsilon}))$ is an odd
function of $\boldsymbol{\epsilon}$ to first order \emph{specifically at} $\mathbf{v}^*$.
For symmetric $p_\epsilon$, contributions from $+\boldsymbol{\epsilon}$ and
$-\boldsymbol{\epsilon}$ cancel exactly:
\[
  \mathbb{E}_\epsilon\!\left[\nabla_{\mathbf{p}} c\bigl(f(\mathbf{v}^*
  + \boldsymbol{\epsilon})\bigr)\right] = \mathbf{0}.
\]
Therefore $\nabla_{\mathbf{v}} J_{\mathrm{aware}}\big|_{\mathbf{v}^*} = \mathbf{0}$,
the same stationarity condition as the noiseless problem.
Note: this argument holds specifically at $\mathbf{v}^*$; for $\mathbf{v} \neq
\mathbf{v}^*$ the cost gradient is not odd in $\boldsymbol{\epsilon}$ and no
cancellation is guaranteed.
The optimal policies coincide; only the expected cost value (variance) differs.
\end{proof}

\begin{remark}
The result relies on linearity of $f$ near the optimum and the zero mean and symmetry of
$\boldsymbol{\epsilon}$.
For skewed or heavy-tailed noise the Taylor approximation breaks and a genuine gap can
emerge.
The biased noise types (Slip, Jitter) have $\mathbb{E}[\boldsymbol{\epsilon}] \neq
\mathbf{0}$ and fall outside this theorem; they are learnable.
\end{remark}

\section{Lengthscale Scaling Rule Derivation}
\label{app:lengthscale}

The RBF policy (Eq.~\ref{eq:policy}) uses a squared-exponential kernel with
lengthscale $\ell_s$.
The sensitivity between two target positions at Euclidean distance $d$ is
\[
  S(d,\, \ell_s) = \exp\!\Bigl(-\tfrac{d^2}{2\ell_s^2}\Bigr).
\]
Config~A (target range $\approx 1.0$\,m, $\ell_s = 1.0$\,m) achieves
$S_A = e^{-0.5} \approx 0.607$.

For a new configuration with target range $R$, preserving the same sensitivity gives
\[
  \exp\!\Bigl(-\frac{R^2}{2\ell_s^2}\Bigr) = e^{-0.5}
  \implies \ell_s = R.
\]
In practice, we use $\ell_s = 0.15\,R$ rather than $R$ because sharper sensitivity
(smaller $\ell_s$) promotes faster convergence when GP data are sparse — each
basis function covers a narrower region, preventing aliasing across nearby targets.
The conservative recommendation is $\ell_s = R$ (preserves Config~A sensitivity);
the empirical finding is that $\ell_s = 0.15\,R$ is safe and typically faster.

\section{Particle Freeze Approximation}
\label{app:freeze}

Once a particle's $z$-coordinate crosses zero, its state is frozen at the last
timestep above ground and the cost is evaluated at that frozen position for all
remaining timesteps.

For $T_s = 0.02$\,s and a typical vertical velocity of 2\,m/s at impact, the true
ground crossing occurs $\Delta t = z_{\mathrm{last}}/v_z \approx 0.01$\,s before the
frozen step, during which the ball travels $\Delta x \approx v_x \cdot \Delta t
\approx 2$--4\,cm horizontally.

In actual rollouts (NumPy and PyBullet), landing is detected and refined by linear
interpolation, so training data are accurate.
The freeze approximation applies only to the MC particles used for the gradient
estimate inside \texttt{apply\_policy}, where per-particle interpolation would break
batched tensor operations.
The resulting cost estimates are slightly pessimistic (policy is penalised for landing
a few centimetres beyond the true crossing), but the bias is consistent across
particles and policy updates, so optimisation converges to the correct policy.
